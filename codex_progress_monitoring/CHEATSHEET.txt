  codex_progress_monitoring — Quick Reference Cheatsheet

PROJECT
  Brightspace progress-monitoring automation for VAU and MAE.

MEMORY
  projectplan.md              Main operating reference
  memory/PROJECT_MEMORY.md    Stable project facts and cautions
  tasks/TASKS.md              Current state
  tasks/DECISIONS.md          Locked decisions
  tasks/ARCHIVE.md            Older session history
  docs/LEGACY_SESSION_NOTES.md Pre-template session notes

SESSION PHRASES
  "start session"             Reload project context and resume work
  "end session"               Record the session, decisions, and next step

------------------------------------------------------------
SCRIPTS
------------------------------------------------------------

  Verify downloaded files:
    python MAE/MAE_0_CheckDownloadedFiles.py
    python VAU/VAU_0_CheckDownloadedFiles.py

  Check duplicates:
    python MAE/MAE_1_CheckAllDups.py
    python VAU/VAU_1_CheckAllDups.py

  Rebuild student maps:
    python MAE/MAE_2_GenerateStudentMap.py
    python VAU/VAU_2_GenerateStudentMap.py

  Generate reports:
    python MAE/MAE_3_AttendanceMissing.py
    python MAE/MAE_4_StrugglingStudents.py
    python MAE/MAE_4_5_PrincipalSummary.py
    python MAE/MAE_5_RemindForBSLogin.py
    python MAE/MAE_6_HighHonoursStudents.py
    python MAE/MAE_7_NeedsToAttendMoreRegularly.py
    python VAU/VAU_3_AttendanceMissing.py
    python VAU/VAU_4_StrugglingStudents.py
    python VAU/VAU_4_5_PrincipalSummary.py
    python VAU/VAU_5_RemindForBSLogin.py
    python VAU/VAU_6_HighHonoursStudents.py
    python VAU/VAU_7_NeedsToAttendMoreRegularly.py

  Supervised runner:
    python -m Common.supervised_runner --campus VAU --flow start --mode silent-test
    python -m Common.supervised_runner --campus VAU --flow main --mode test-send --week 29
    python -m Common.supervised_runner --campus VAU --step 4_5 --mode test-send --week 29
    python -m Common.supervised_runner --campus MAE --flow start --mode silent-test

------------------------------------------------------------
NOTES
------------------------------------------------------------

  - Run scripts from the project root.
  - When running a campus pipeline conversationally, step `*_0_CheckDownloadedFiles` runs first; choose the week number only after that step passes.
  - Outlook COM automation requires Windows Outlook to be configured.
  - pdfkit requires wkhtmltopdf to be installed locally.
  - Reports continue to write to Ready For Printing/, not output/.
